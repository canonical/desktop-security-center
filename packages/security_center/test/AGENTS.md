# Test structure

- **Page tests** (`app_permissions/`, `disk_encryption/`, `ubuntu_pro/`) — widget tests with mocked services
- **Service tests** (`services/`) — unit tests, no widget apparatus
- **Shared utilities** — @packages/security_center/test/test_utils.dart and @packages/security_center/test/utils/ubuntu_pro_utils.dart

# Mock generation

Mocks are generated by **mockito** via `build_runner`. The `@GenerateMocks` annotations live on helper functions in `test_utils.dart`. After changing mock annotations, run `melos generate` from the repo root.

# Writing page tests

```dart
testWidgets('description', (tester) async {
  final container = createContainer();
  registerMockXxxService(...);        // parameterized mock factories from test_utils
  registerMockLocalSnapData();

  await tester.pumpApp(
    (_) => UncontrolledProviderScope(
      container: container,
      child: const MyPage(),
    ),
  );
  await tester.pump();

  // interact and assert
  await tester.tap(find.byType(SomeWidget));
  verify(mockService.someMethod()).called(1);
});
```

Key helpers on `WidgetTester`: `pumpApp(builder)`, `context`, `l10n` (for localized assertion strings).

Services are registered with `registerMockService<T>(mock)` and torn down with `addTearDown(unregisterService<T>)`.

# Writing service tests

Pure unit tests — instantiate the service directly with mock dependencies (e.g., `MemoryFileSystem`, mock process runners). No Riverpod or widget setup needed.

# Mock factory functions

`test_utils.dart` provides parameterized mock factories for each service (e.g., `registerMockDiskEncryptionService(checkRecoveryKey:, checkError:, ...)`). These pre-stub common return values so tests only configure what they care about.

Ubuntu Pro tests have additional factories in `test/utils/ubuntu_pro_utils.dart` covering D-Bus mocking, HTTP mock servers, and filesystem fixtures.
